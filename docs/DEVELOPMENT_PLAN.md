# 开发流程规划

## 阶段 1：需求拆解与边界定义

- 明确角色：学生、学院管理员、系统管理员
- 明确对象：学生、活动、学分规则、学分记录
- 明确输出：个人统计、分类统计、时间维度统计

## 阶段 2：技术方案设计

- 后端采用 Spring Boot + JPA + MySQL
- API 风格统一为 RESTful
- 统一返回结构与统一异常处理

## 阶段 3：数据模型设计

- 设计学生、活动、学分规则、学分记录四类核心实体
- 约束唯一字段（如学号、规则分类）
- 定义审批状态字段，支撑后续流程扩展

## 阶段 4：核心功能开发

- 学生管理模块
- 活动管理模块
- 学分申请模块
- 学分汇总统计模块

## 阶段 5：验证与质量保障

- 编译验证与接口联调
- 核心业务分支补充测试
- 异常路径验证（参数非法、数据不存在）

## 阶段 6：文档与交付

- 输出技术选型说明
- 输出数据库初始化说明
- 推送到 Git 仓库并持续迭代
