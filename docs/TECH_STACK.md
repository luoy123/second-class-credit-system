# 技术栈说明

## 后端框架

- **Spring Boot 3.2.5**
    - 快速构建 RESTful 服务
    - 内置自动配置，降低基础设施代码

## 前端框架（前后端分离）

- **Vue 3 + Vite**
    - 构建独立管理端单页应用
    - 通过 Vite 开发代理将 `/api` 转发到后端 `127.0.0.1:8080`
- **Vue Router 4**
    - 管理登录与业务页面路由
    - 路由守卫控制未登录访问
- **Axios**
    - 统一 API 请求封装与 JWT 令牌注入
    - 统一处理后端 `ApiResponse(code/message/data)` 响应格式

## 持久化与数据库

- **Spring Data JPA**
    - 使用仓储接口简化 CRUD
    - 降低 SQL 样板代码
- **MySQL 8+**
    - 结构化数据存储
    - 适合学分、活动、统计等关系型场景

## 校验与开发效率

- **Spring Validation**
    - 统一请求参数校验
    - 在入口层保障数据合法性
- **JJWT**
    - 生成与校验 JWT 访问令牌
    - 支撑管理员接口的 Bearer Token 鉴权
- **Lombok**
    - 简化实体与 DTO 模板代码
    - 提升开发效率

## 构建与运行

- **Maven**
    - 依赖管理与构建打包
- **Java 17**
    - 长期支持版本，兼容 Spring Boot 3.x
- **Node.js 20+（推荐）**
    - 支撑 Vue 前端构建与开发调试

## 当前架构原则实践

- **KISS**：仅实现学分统计核心主链路
- **YAGNI**：暂不引入消息队列、缓存、分布式事务
- **DRY**：统一响应结构与异常处理，避免重复控制器代码
- **SOLID**：控制器、服务、仓储职责清晰分层
